\--- 以下为译文（我通读一遍后觉得不甚好理解的地方有作些改动） ---

OpenClaw（前身为 Clawdbot，后更名为 Moltbot）是一个开源 AI 代理，它不仅能回答问题，还能为你执行任务。

当你将其连接到日历、电子邮件、文件系统或终端等工具时，它可以创建文件、发送消息、运行命令和更新系统，无需你每次都亲自介入。

人们使用 OpenClaw 来处理日常管理工作、支持开发者工作流程，以及管理那些否则需要手动检查的长期任务。

OpenClaw 可以在个人设备（如台式机或 Mac mini）上运行，但这意味着你需要让设备保持开机并连接到互联网。为了实现 24/7 的可靠性，如定时晨间简报、持续监控或在你睡眠时运行的自动化，将其托管在服务器上更合理。

在你自己的服务器上运行 OpenClaw，可以让你完全掌控数据、集成和正常运行时间，因为没有任何东西需要通过第三方服务。

但这种程度的访问权限也伴随着责任。如果配置不当，拥有系统权限和 API 密钥的代理可能会造成实际问题，因此防护措施和限制与灵活性同样重要。

### 1\. 获取 2 分钟晨间简报

2 分钟晨间简报会在你查看手机之前，通知你天气、前几个日历事件和头条新闻。它帮助你在不打开多个应用的情况下，以知情的状态开始新的一天。

你可以安排 OpenClaw 每天同一时间发送此摘要。使用 cron 任务或 OpenClaw 的内置调度器，在每天早上 6:30 触发提示。当它在虚拟专用服务器（VPS）上运行时，代理会全天候在线。

它可以从天气 API、日历集成以及 RSS 源或新闻 API 提取数据，然后将所有内容格式化为简短的消息。

示例提示： 每天早上 6:30，给我发送一条包含以下内容的消息：

  * [你的城市]的今日天气预报
  * 我的前三个日历事件
  * BBC 新闻的前三个头条 保持在 150 字以内。 



### 2\. 从聊天消息构建共享购物清单

共享购物清单从聊天消息中收集杂货项目，并将它们保存在一个可以在商店打开的地方。

当你的家庭成员发送"我们需要牛奶"或"多拿点鸡蛋"时，OpenClaw 会自动将这些项目添加到共享文档或数据库中。

此设置解决了一个常见的家庭问题：杂货提醒在忙碌的聊天中被淹没，几天后没人记得之前提到了什么。

OpenClaw 监控关键词，提取项目名称，并在更新列表时去重。

当你在商店时，向 OpenClaw 询问当前列表。它会返回按类别分组的一切内容，如乳制品、农产品和储藏室食品，这样你可以更快地在过道中穿行。

### 3\. 将语音笔记转换为每日日记条目

将语音笔记转换为每日日记条目，让你可以在一天中捕捉快速的想法。这些可能是在通勤时的反思、会议后的想法或睡前的观察。

OpenClaw 转录这些录音，并在每晚将其整理成整洁的日记条目。

工作流程从语音转文本开始，将你的音频转换。如果录音格式不受支持，可以在转录前在服务器上安装 ffmpeg。

接下来，代理将原始转录整理成清晰的部分，如情绪、亮点、学到的教训和明天的重点。你可以查看和调整条目，或者让 OpenClaw 自动将其保存到你的笔记应用中。

### 4\. 转录会议并提取行动项目

转录会议并提取行动项目可以让你清楚地了解发生了什么。你无需重新观看录音或解读混乱的笔记，就能看到决策、分配的任务、负责人和截止日期。

首先上传会议录音，在几分钟内获得结构化输出。OpenClaw 然后转录音频，尽可能识别发言人，并从对话中提取关键细节。

输出包括关键时刻的时间线和一个单独的行动项目列表，你可以直接发送到任务管理器。

示例提示： 转录此会议录音。然后创建：

  1. 主要讨论点的三段总结
  2. 行动项目列表，每个都有负责人和截止日期
  3. 最终决策列表



### 5\. 自动跟踪包裹和配送状态

自动包裹跟踪从订单确认电子邮件或运输标签照片中提取跟踪号码。OpenClaw 检查承运商 API，并维护一个显示所有收货配送状态的简单仪表板。

你可以为"正在配送"和"延误"更新设置警报。这样你就知道何时可以期待包裹，或者何时需要跟进问题。

如果你使用专用的商务电子邮件转发订单确认和运输通知，此功能效果最佳。对于个人使用，你可以转发个别电子邮件或设置自动转发规则。（这一点现在各个小程序是都可以做到的。）

### 6\. 总结未读邮件并更快达到收件箱零目标

邮件摘要为你提供未读消息的每日摘要。它突出显示紧急内容，标记需要回复的消息，并建议回复草稿，以便你能更快达到收件箱零。

OpenClaw 扫描主题行、发送者模式和消息内容，以突出重要内容并将无关内容推到一边。

摘要可能如下所示："今天需要回复的三项紧急事项，七条仅供参考的消息，以及十二条可以安全归档的促销邮件。"

对于高优先级消息，代理可以起草一个你可以原样发送或先编辑的回复。

在使用 OpenClaw 设置邮件摘要时，首先从有限的范围开始，例如单个标签或文件夹，然后再授予更广泛的访问权限。

此外，先在低风险消息上测试。在信任代理处理主收件箱之前，查看它对邮件分类的准确性。

### 7\. 监控 X 上的品牌提及并发送每日报告

品牌提及监控跟踪人们在 X（前 Twitter）上对你的品牌、产品或名称的评论。它让你免于手动搜索，并将关键对话汇总到单个报告中。

OpenClaw 搜索平台，过滤噪音，并按照你选择的时间表提供情感分析。

报告突出显示整体情感，指出有影响力的账户，并标记需要回应的帖子。你可以在产品发布期间每小时安排一次，或为了持续监控每天安排一次。

示例提示：

每天上午 9 点，搜索 X 上过去 24 小时内提及"[你的品牌名称]"的内容。总结：

  * 总提及次数
  * 整体情感细分（正面、中性、负面）
  * 提及我们的前三个参与度最高的帖子
  * 任何需要注意的投诉或支持请求



### 8\. 自动化新客户入职任务

自动化客户入职在新客户签约时启动完整的工作流程。它创建项目文件夹，发送包含后续步骤的欢迎邮件，安排启动电话，并将跟进提醒添加到任务列表。

每个客户都获得相同的一致体验，无需手动复制模板或依赖人工记忆。

你只需定义一次工作流程，然后通过告诉 OpenClaw 新客户的姓名和电子邮件来触发它。

代理在 Google Drive 或 Dropbox 中创建文件夹，发送模板化邮件，并根据你的时间线添加日历事件或任务。

### 9\. 将收据转换为费用电子表格条目

收据转电子表格自动化从收据照片中提取供应商名称、日期、金额和类别。然后它将每个条目记录在与你的费用报告格式匹配的电子表格中。

你可以拍摄收据照片，将其发送给 OpenClaw，数据会自动出现在正确的列中。

设置此功能时，提前标准化你的类别，如旅行、软件、广告和办公用品。这有助于代理一致地分类费用。

随着时间的推移，它会学习你的模式，因此来自同一供应商的收据总是落在正确的类别中。

### 10\. 按计划将 KPI 快照发送到 Slack 或 Discord

定时 KPI 快照从仪表板捕获关键指标，并在每个周一早上将其发布到团队频道。

这为你提供了轻量级报告，而无需构建自定义仪表板或向每个人提供访问分析工具的权限。

OpenClaw 使用浏览器自动化登录、打开正确视图、截取屏幕截图，并将其与摘要一起发布到 Slack 或 Discord。

它与 Google Analytics、Stripe 和内部仪表板等工具集成。如果工具具有 Web 界面并允许自动访问，OpenClaw 通常可以捕获数据。

### 11\. 头脑风暴内容创意以克服写作障碍

内容头脑风暴根据你想要填补的目标、受众或内容差距，帮助你生成主题创意、大纲、钩子和角度。

你不必盯着空白页面，而是获得可以反应和细化的原始材料。当你卡住时，OpenClaw 为你提供可靠的起点。

示例提示： 我为学习网络托管的小企业主写作。给我：

  * 10 篇他们会觉得有用的博客文章创意
  * 该列表中最好的标题及其有效的原因
  * 该首选的五点大纲



### 12\. 从简单大纲生成初稿

初稿生成将要点转化为文章、电子邮件或着陆页的结构化草稿。它将关键想法扩展为带有过渡、示例和清晰流程的完整段落。

首先分享你的大纲，然后让 OpenClaw 处理扩展。

添加一些约束条件以提高输出质量。例如，指定语气（对话式、正式或技术性）、目标字数以及诸如"包含具体示例"或"避免营销废话"等规则。

结果是一个你可以编辑和改进的草稿，而不是从空白页面开始。

### 13\. 无需打开设计工具即可生成符合品牌形象的图像

符合品牌形象的图像生成允许你使用清晰的风格说明（如调色板、布局和情绪）创建社交媒体视觉效果。OpenClaw 通过连接到外部图像 API 的技能生成图像。

捆绑的 nano-banana-pro 技能使用 Google 的 Gemini 图像模型，而 openai-image-gen 连接到 OpenAI 的图像 API，包括 DALL·E。社区技能添加更多选项，如 Fal.ai 和 Replicate。

要获得一致的结果，你需要了解如何使用 AI 图像生成器。明确的约束和具体细节有助于生成符合你品牌形象的图像并减少手动编辑。

示例提示：

创建一个社交媒体图像（1200 × 630 像素），包含：

  * 深蓝色渐变背景
  * 白色文字，显示"每个 VPS 所有者都需要的 5 个服务器安全技巧"
  * 极简主义技术或安全图标
  * 适合 LinkedIn 的简洁现代风格 保存到 /images/social/



### 14\. 将一篇文章重新用于多平台内容

内容重用将单篇博客文章转换为 X Thread、LinkedIn 帖子、Instagram 标题和电子邮件时事通讯片段。它将核心信息调整到每个平台的格式、长度限制和受众期望。

每个版本在保持消息一致性的同时改变交付方式。

要开始，分享原始文章或 URL 供 OpenClaw 获取。代理然后自动创建特定平台的版本。

例如，X Thread将关键点分解为具有强钩子的短帖子。LinkedIn 添加专业背景。Instagram 的标题更短更有力。TikTok 脚本专注于你可以在镜头前传达的快速要点。

### 15\. 在论坛和评论中快速起草常见问题的回复

快速回复起草帮助你在 Discord 服务器、论坛线程或社交媒体评论中快速回答常见问题。

它涵盖价格问题、运输询问、退款政策和基本产品详细信息，全部使用友好、符合品牌形象的语言。

设置此功能时，OpenClaw 为你创建草稿回复，以便在发送前查看。

首先准备常见场景的模板。然后将代理指向新消息并获得草稿回复。你仍然控制最终响应。

### 16\. 从聊天运行安全的 Shell 命令以进行快速修复

从聊天运行安全的 Shell 命令让你无需打开终端即可处理常见的服务器任务。你可以直接从聊天界面检查磁盘使用情况、重启服务、查看最近的日志或清除缓存。

用自然语言输入你想要的内容。OpenClaw 将其翻译为 Shell 命令，运行它，并在聊天中确认执行。

这对于你经常运行的 Linux 命令效果最佳，例如检查服务器负载或查看最近的日志。对于快速、重复的任务，在聊天中输入请求比打开 PuTTY 等 SSH 客户端更快。

设置此功能时，请专注于安全和限制：

  * 保留允许命令的允许列表。
  * 以受限权限作为非 root 用户运行代理。
  * 记录所有执行的命令以进行审计。
  * 切勿在代理配置中存储 root 凭据。



### 17\. 监控服务器健康状况并在出现问题时发出警报

带有阈值警报的服务器健康监控让你在出现问题时知道。

例如，当磁盘使用率超过 85%、CPU 使用率超过 80% 超过五分钟，或 RAM 使用率超过 90% 时，你可以收到警报。这让你有足够的时间在服务崩溃前修复问题。

你可以根据工作负载设置不同的阈值。生产系统通常需要更严格的限制，而开发环境可以更灵活。

这对于运行长期自动化、数据库或 Web 应用程序的 VPS 用户很重要。已满的磁盘或内存泄漏可能会毫无预警地导致服务关闭。

无需在随机时间检查 Linux 磁盘空间或系统负载，OpenClaw 为你监控这些指标，仅在需要采取行动时才发出警报。

### 18\. 监控 CI/CD 管道并在失败时发出通知

CI/CD 管道监控在构建失败、测试出错或部署完成时向你发出警报。它突出来自 GitHub Actions、GitLab CI 或 Jenkins 的重要状态更改，因此你不必整天查看仪表板。

你可以在首选聊天频道中收到有关失败和完成的通知。

示例提示： 监控我的 GitHub Actions 工作流程。当以下情况时通知我：

  * 任何工作流程失败
  * 生产部署完成，无论成功还是失败
  * 测试套件花费时间超过 10 分钟 包含提交消息和失败运行的链接。 如果你的 CI/CD 管道也构建和推送容器镜像，使用私有 Docker 注册表有助于保持这些工件的组织性和安全性。



### 19\. 总结拉取请求以加速代码审查

拉取请求总结让你清楚地了解代码更新中更改了什么。

它包括高级摘要、最大添加或删除的文件列表，以及潜在问题的草稿反馈。这可以显著加快代码审查的速度。

OpenClaw 读取差异，标记有风险的模式，并建议你可能想问作者的问题。你可以使用它来快速分类拉取请求，并决定哪些需要更深入的审查，哪些可能可以安全批准。

虽然代理加快了这一过程，但它不能取代对代码质量或架构决策的人工判断。

如果你是拉取请求或版本控制的新手，了解什么是 GitHub及其如何融入现代开发工作流程会有所帮助。从那里开始，学习如何在 Ubuntu 上安装 Git，使审查拉取请求更容易。

### 20\. 发现过时的依赖项并计划安全的升级

依赖扫描使用 OpenClaw 检查项目中的过时软件包，突出显示与安全相关的更新，并标记潜在的破坏性更改。

将代理指向你的 package.json、requirements.txt 或 Gemfile，它会返回升级建议的优先级列表。

OpenClaw 还可以生成逐步升级清单。

这包括备份当前状态，首先在暂存环境中应用更新，验证没有任何破坏，并准备好回滚计划部署到生产环境。

该过程有助于你避免有风险的"更新并祈祷"方法，这种方法经常导致停机。

请注意，依赖管理只是整体的一部分。我们的VPS 安全技巧涵盖了有助于保护基础设施其余部分的服务器加固实践。

### 21\. 研究产品并编写简短的比较报告

产品研究收集网络搜索结果，根据你的标准比较选项，并返回结构化摘要。它处理初步研究，以便你可以专注于最终决策。

在评估工具、服务或产品时，OpenClaw 以一致的格式呈现发现。

示例输出格式：

最佳选项：[产品名称] 价格：[成本和计费模式] 主要优势：[两到三个主要优点] 权衡：[需要考虑的局限性] 最适合：[理想用户或用例]

### 22\. 通过单个聊天命令控制智能家居设备

智能家居控制允许你通过 Webhook、IFTTT 或 Home Assistant 集成，使用单个聊天命令触发灯光、智能插头或例程。

你可以发送诸如"打开办公室灯光"或"关闭加热器"之类的消息，OpenClaw 会发送相应的 API 调用。

这对于简单的"现在执行此操作"命令和预定例程效果最佳，而不是复杂的条件逻辑，如"如果 X 发生，则执行 Y 而不是 Z"。

对于更高级的家庭自动化，将 OpenClaw 连接到本地 Home Assistant 实例。设置 VPS并在那里运行两者，为你提供可控制的专用环境。

### 23\. 获取食谱创意并制定每周膳食计划

使用 OpenClaw 进行食谱和膳食规划会根据你拥有的食材、饮食限制或烹饪偏好建议餐食。然后它构建每周计划并创建组合购物清单。

你可以向 OpenClaw 询问想法，选择你的最爱，并导出购物清单。

这可以与用例 [#2](javascript:;) 相互配合：膳食规划输出可以直接输入到你的共享购物清单，这样在规划和购物之间不会遗漏任何东西。

### 25\. 自动化浏览器任务如表单填写和管理工作

浏览器自动化允许你登录到 Web 工具、在页面之间复制数据并提交重复表单，而无需手动点击。当没有可用的 API 时，它对于管理任务很有用。

OpenClaw 控制无头浏览器来导航界面、填写字段并自动触发操作。

这对于在平台上更新产品信息、向多个服务提交相同的表单，或从仅提供用户界面的 Web 应用程序中提取数据等任务效果良好。

也就是说，浏览器自动化是需要启用的风险较高的功能之一。网页可能包含利用提示注入的隐藏指令。

恶意或受损的网站可能会欺骗 OpenClaw 泄漏数据或采取意外操作。将此功能限制为你控制的内部工具，例如管理仪表板或内联网应用程序，而不是公共网站。

对于涉及付款、帐户更改或敏感数据的任何内容，我们强烈建议完全避免浏览器自动化。

即使使用受信任的工具，也要预期一些实际限制：

• CAPTCHA 可能会阻止自动访问。 

• 会话可能会过期并需要重新身份验证。

• 速率限制可能会减慢重复请求。

如果你在同一台服务器上运行多个内部工具，配置 NGINX 作为反向代理有助于在服务之间路由流量并保持自动化目标的组织性。

如何在 VPS 上安全地运行 OpenClaw？

要在 VPS 上安全地运行 OpenClaw，请遵循最小权限原则、限制允许的命令、谨慎管理机密并隔离代理环境。

像 OpenClaw 这样的工具具有 Shell 访问权限、API 密钥和浏览器控制。如果配置不当，它可能会造成严重损害，因此这些基础知识很重要：

  * 最小权限。 以非 root 用户身份运行代理，仅具有必要的权限。如果只需要日历访问权限，请不要授予 Shell 访问权限。
  * 受限命令。 保留明确批准的 Shell 命令的允许列表。阻止破坏性操作，如 rm -rf、dd 或任何触及系统文件的操作。
  * 机密管理。 在环境变量或机密管理器中存储 API 密钥和凭据。切勿将它们硬编码在配置文件或对话历史记录中。
  * 隔离环境。 在容器、虚拟或本地环境中运行代理，以在出现问题时减少爆炸半径。



对于额外的服务器加固，还要应用这些防护措施：

  * 单独的用户帐户。 为 OpenClaw 创建专用的系统用户，具有最少的组成员资格。
  * SSH 加固。 禁用密码身份验证，仅使用基于密钥的访问，并限制可以通过 SSH 连接的用户。
  * 有限的集成。 从一两个集成开始，确认它们安全运行，然后扩展。不要同时连接电子邮件、日历、文件系统和 Shell 访问权限。
  * 详细日志记录。 为所有代理操作启用详细日志记录，以便你可以查看发生了什么以及何时发生。
