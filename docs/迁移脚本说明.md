# 微信公众号 → Hugo 站点：脚本执行说明

在**仓库根目录**执行下文命令（路径以 `wygmjdd.github.io` 为例，请换成你的本机路径）。不要在 `scripts/` 子目录里跑 `python3 -m scripts.wechat.*`，否则会出现 `ModuleNotFoundError: No module named 'scripts'`。

## 环境与依赖

```bash
cd /path/to/wygmjdd.github.io
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r scripts/requirements.txt
playwright install chromium # 抓取/下载图片等会用到浏览器内核
```

## 流水线总览（三步 + 可选整理）

| 步骤 | 作用 | 模块 |
|------|------|------|
| 1 | 从专辑配置生成占位 `_posts`，或一次性拉取全文到 `_posts` | `scripts.wechat.migrate` |
| 2 | 按 `source_url` 拉取正文、下载图片 → `_rehydrated_posts` | `scripts.wechat.rehydrate_posts` |
| 3 | 把再水合后的 Markdown 生成 Hugo 的 `content/docs/` | `scripts.wechat.migrate_jekyll_to_hugo_book` |
| 可选 A | 统一文末「原文链接」、去掉公众号推广尾句 | `scripts.wechat.normalize_article_footer` |
| 可选 B | 把 `![图](url)说明` 整行改成带样式的 `<figure>` 说明 | `scripts.wechat.postprocess_image_captions` |

重要配置路径：

- 专辑列表：`_archive/legacy-jekyll/_data/wechat_albums.yml`
- Hugo 侧栏分类：仓库根目录 `data/categories.yml`（不是 `_archive` 里那份）
- 第二步默认读：`_archive/legacy-jekyll/_posts`，写：`_archive/legacy-jekyll/_rehydrated_posts`

---

## 第一步：`migrate`（专辑 → `_posts`）

**配置**：在 `wechat_albums.yml` 里填好各专辑 `url`（或用手动 URL 列表，见 `scripts/wechat/MANUAL_URLS.md`）。

**只生成占位 stub（推荐与第二步拆开跑）**：

```bash
python3 -m scripts.wechat.migrate --stubs-only
```

**全量抓取**（直接写入完整文章到 `_posts`，与 stub 模式不要对同一批 URL 混用以免重复文件）：

```bash
python3 -m scripts.wechat.migrate
python3 -m scripts.wechat.migrate --resume   # 跳过已在 _posts 或 scripts/wechat/.migrate_done 里出现过的 URL
```

**常用参数**：

- `--dry-run`：只打印，不写文件  
- `--no-playwright`：不用浏览器；专辑页若依赖 Playwright 则可能没有 URL

## 手动批次更新：`update_manual`

### 从 Chrome 历史生成批次 JSON（推荐）

在 Chrome（默认 **Chrome Beta**）里打开要搬运的公众号文章后，可用历史记录自动生成批次文件：

```bash
python3 -m scripts.wechat.update_manual.collect_from_chrome
```

默认行为：

- 读取 `~/Library/Application Support/Google/Chrome Beta/Default/History`（稳定版用 `--channel stable`）
- 只采集**今天**访问过的 `mp.weixin.qq.com/s` 链接（`--batch-id` 默认今天日期）
- 排除 `content/docs/` 里已有 `source_url` 的文章
- 全部写入同一 slug，默认 `subway-diary`；JSON 含 Chrome 页面标题 `title`，跑 `update_manual` 前请按标题核对并修改每条 `slug`
- 输出到 `scripts/wechat/update_manual/input/<batch-id>.json`（gitignore）

生成 JSON 示例：

```json
[
  {
    "title": "文章标题（来自 Chrome 历史）",
    "slug": "subway-diary",
    "url": "https://mp.weixin.qq.com/s/..."
  }
]
```

常用参数：

```bash
# 预览，不写文件
python3 -m scripts.wechat.update_manual.collect_from_chrome --dry-run

# 回溯最近 30 天
python3 -m scripts.wechat.update_manual.collect_from_chrome --days 30

# 改用稳定版 Chrome
python3 -m scripts.wechat.update_manual.collect_from_chrome --channel stable

# 指定其它分类
python3 -m scripts.wechat.update_manual.collect_from_chrome --default-slug reading-category
```

生成 JSON 后，再跑（`<batch-id>` 与上面一致，默认即今天）：

```bash
python3 -m scripts.wechat.update_manual
```

---

当你已经手写好某个批次文件，例如
`scripts/wechat/update_manual/input/2026-05-26.json`，可以直接用这个入口跑完整更新流程。

在仓库根目录执行：

```bash
cd /path/to/wygmjdd.github.io
source .venv/bin/activate
python3 -m scripts.wechat.update_manual 2026-05-26
```

这里的参数是批次 ID，也就是 JSON 文件名去掉 `.json` 后的部分。

例如：

- `2026-05-26.json` -> `python3 -m scripts.wechat.update_manual 2026-05-26`
- `2026-04-11.json` -> `python3 -m scripts.wechat.update_manual 2026-04-11`

常用可选参数：

```bash
python3 -m scripts.wechat.update_manual 2026-05-26 --no-skip-existing-rehydrated
```

默认情况下，`update_manual` 在 rehydrate 阶段会附带 `--skip-if-output-exists`；加上
`--no-skip-existing-rehydrated` 后，会改为不跳过已存在的再水合结果。

这个命令会顺带完成以下步骤：

1. 合并 `scripts/wechat/update_manual/input/<batch>.json` 到 `_archive/legacy-jekyll/_data/wechat_manual_article_urls.yml`
2. 运行 `scripts.wechat.migrate --stubs-only --no-playwright`
3. 只对当前批次对应的 stub 跑 `scripts.wechat.rehydrate_posts`
4. 重建 `content/docs/`
5. 运行 `normalize_article_footer` 和 `postprocess_image_captions`

注意：这个入口也会重建 `content/docs/`，所以执行前请确认当前站点内容改动是否已备份或已提交。

## 第二步：`rehydrate_posts`（`_posts` → `_rehydrated_posts`）

从 stub（或已有 front matter）里的 `source_url` 拉微信正文，并把图片落到 `static/images/wechat/...`，正文里改为本地路径。

**整目录批量（默认目录）**：

```bash
python3 -m scripts.wechat.rehydrate_posts
```

**只测一两篇**：

```bash
python3 -m scripts.wechat.rehydrate_posts \
  --file _archive/legacy-jekyll/_posts/2033-01-01-post-xxxx.md \
  --file _archive/legacy-jekyll/_posts/2033-01-01-post-yyyy.md
```

**跳过记录去重后，只重试失败篇目**：

```bash
# 仅重写 skipped 日志（按 canonical source_url 去重）
python3 -m scripts.wechat.rehydrate_posts --dedupe-skipped-log-only scripts/wechat/rehydrate_skipped.yml

# 按日志里的路径只跑这些文件
python3 -m scripts.wechat.rehydrate_posts --from-skipped-log scripts/wechat/rehydrate_skipped.yml
```

**节流（防请求过快）**：脚本内置篇与篇之间的随机间隔、以及每批后的长休息。可调参数示例：

- `--between-min` / `--between-max`：两篇之间的睡眠秒数（默认约 2～6）
- `--batch-min` / `--batch-max`：多少篇后进入长休息（默认约 3～5）
- `--sleep-min` / `--sleep-max`：长休息秒数（默认约 180～600）

**输出与日志**：

- `--out-dir`：再水合输出目录（默认 `_archive/legacy-jekyll/_rehydrated_posts`）
- `--skipped-log`：失败记录 YAML（默认 `scripts/wechat/rehydrate_skipped.yml`）

## 第三步：`migrate_jekyll_to_hugo_book`（→ `content/docs`）

从 `_rehydrated_posts`（若为空则回退读 `_posts`）生成 Hugo 内容。

```bash
python3 -m scripts.wechat.migrate_jekyll_to_hugo_book
```

**注意**：

- 运行时会**删除并重建**整个 `content/docs/`。执行前请确认已备份或已提交 Git。
- 同一篇 `source_url` 在源数据里若对应多个文件，脚本会**合并为一篇**再输出。
- 若要从别的目录读入文章，可设置环境变量：  
  `WYGMJDD_POSTS_DIR=/绝对路径/到/markdown目录 python3 -m scripts.wechat.migrate_jekyll_to_hugo_book`

处理已生成在 Hugo 下的 Markdown（默认根目录为 `content/docs`）：

python3 -m scripts.wechat.normalize_article_footer
python3 -m scripts.wechat.normalize_article_footer --dry-run

## 可选：`postprocess_image_captions`（图片说明样式）

把整行形式的 `![说明](url)图片下方说明文字` 转为 `<figure class="figure-with-caption">`，样式在 `assets/_custom.scss`。依赖 Hugo 配置里 Goldmark `unsafe = true`（本仓库已开）。

```bash
python3 -m scripts.wechat.postprocess_image_captions
python3 -m scripts.wechat.postprocess_image_captions --dry-run
python3 -m scripts.wechat.postprocess_image_captions content/docs/2026/01/summary__post-xxx.md
```

建议在第三步（及如有 `normalize_article_footer`）之后再跑，避免后续生成覆盖手工格式。

## 小红书单篇文章出图（article series）

在 Cursor 中触发 Skill **xhs-article-cards**（或说「生成小红书图」+ 文章路径），按 Skill 流程选标题、生成封面、写 manifest 后渲染：

```bash
python3 -m scripts.xhs.generate_xhs_cards \
  --series article \
  --manifest scripts/xhs/output/articles/<slug>/manifest.json
```

- 脚本与 Skill 已入库；仅 `scripts/xhs/output/` 为本地生成物（gitignore）
- **每台机器一次性准备**（venv + 依赖 + Playwright）；命令请用 `source .venv/bin/activate` 或 `.venv/bin/python3`
- 需 `playwright install chromium`
- 设计说明：`docs/superpowers/specs/2026-06-27-xhs-article-cards-design.md`

## 常见问题

1. **第二步很慢**：属正常；日志里会有进度与休眠提示。不要随意把 `--between-*`、`--sleep-*` 调得过小，以免触发风控。  
2. **图片仍打不开**：确认已 `playwright install`，且抓取逻辑会优先 `requests`、失败再用 Playwright。  
3. **侧栏分类不对**：改仓库根目录 `data/categories.yml`，与 `migrate_jekyll_to_hugo_book` 使用的数据源一致。  
4. **`content/docs` 被第三步清空**：迁移前务必备份或提交；仅改几篇时考虑只改单文件，而不是重跑第三步。  
5. **`--resume` 不生效 / 重复抓取**：确认进度文件在 `scripts/wechat/.migrate_done`（旧路径 `scripts/.migrate_done` 需手动挪过来）。  
6. **小红书 article 出图失败**：确认 `playwright install chromium`；manifest 中 `cover-bg.png` 存在；正文页过多时可把 manifest 的 `chars_per_slide` 调高（默认 270）。

文档随仓库脚本变更可能过时；若参数与 `python3 -m scripts.wechat.<module> --help` 不一致，以模块为准。
