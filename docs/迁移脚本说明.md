# 微信公众号 → Hugo 站点：脚本执行说明

在**仓库根目录**执行下文命令（路径以 `wygmjdd.github.io` 为例，请换成你的本机路径）。

## 环境与依赖

```bash
cd /path/to/wygmjdd.github.io
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r scripts/requirements.txt
playwright install chromium # 抓取/下载图片等会用到浏览器内核
```

## 流水线总览（三步 + 可选整理）

| 步骤 | 作用 | 脚本 |
|------|------|------|
| 1 | 从专辑配置生成占位 `_posts`，或一次性拉取全文到 `_posts` | `scripts/migrate.py` |
| 2 | 按 `source_url` 拉取正文、下载图片 → `_rehydrated_posts` | `scripts/rehydrate_posts.py` |
| 3 | 把再水合后的 Markdown 生成 Hugo 的 `content/docs/` | `scripts/migrate_jekyll_to_hugo_book.py` |
| 可选 A | 统一文末「原文链接」、去掉公众号推广尾句 | `scripts/normalize_article_footer.py` |
| 可选 B | 把 `![图](url)说明` 整行改成带样式的 `<figure>` 说明 | `scripts/postprocess_image_captions.py` |

重要配置路径：

- 专辑列表：`_archive/legacy-jekyll/_data/wechat_albums.yml`
- Hugo 侧栏分类：仓库根目录 `data/categories.yml`（不是 `_archive` 里那份）
- 第二步默认读：`_archive/legacy-jekyll/_posts`，写：`_archive/legacy-jekyll/_rehydrated_posts`

---

## 第一步：`migrate.py`（专辑 → `_posts`）

**配置**：在 `wechat_albums.yml` 里填好各专辑 `url`（或用手动 URL 列表，见脚本内说明）。

**只生成占位 stub（推荐与第二步拆开跑）**：

```bash
python3 scripts/migrate.py --stubs-only
```

**全量抓取**（直接写入完整文章到 `_posts`，与 stub 模式不要对同一批 URL 混用以免重复文件）：

```bash
python3 scripts/migrate.py
python3 scripts/migrate.py --resume   # 跳过已在 _posts 或 scripts/.migrate_done 里出现过的 URL
```

**常用参数**：

- `--dry-run`：只打印，不写文件  
- `--no-playwright`：不用浏览器；专辑页若依赖 Playwright 则可能没有 URL  

---

## 第二步：`rehydrate_posts.py`（`_posts` → `_rehydrated_posts`）

从 stub（或已有 front matter）里的 `source_url` 拉微信正文，并把图片落到 `static/images/wechat/...`，正文里改为本地路径。

**整目录批量（默认目录）**：

```bash
python3 scripts/rehydrate_posts.py
```

**只测一两篇**：

```bash
python3 scripts/rehydrate_posts.py \
  --file _archive/legacy-jekyll/_posts/2033-01-01-post-xxxx.md \
  --file _archive/legacy-jekyll/_posts/2033-01-01-post-yyyy.md
```

**跳过记录去重后，只重试失败篇目**：

```bash
# 仅重写 skipped 日志（按 canonical source_url 去重）
python3 scripts/rehydrate_posts.py --dedupe-skipped-log-only scripts/rehydrate_skipped.yml

# 按日志里的路径只跑这些文件
python3 scripts/rehydrate_posts.py --from-skipped-log scripts/rehydrate_skipped.yml
```

**节流（防请求过快）**：脚本内置篇与篇之间的随机间隔、以及每批后的长休息。可调参数示例：

- `--between-min` / `--between-max`：两篇之间的睡眠秒数（默认约 2～6）
- `--batch-min` / `--batch-max`：多少篇后进入长休息（默认约 3～5）
- `--sleep-min` / `--sleep-max`：长休息秒数（默认约 180～600）

**输出与日志**：

- `--out-dir`：再水合输出目录（默认 `_archive/legacy-jekyll/_rehydrated_posts`）
- `--skipped-log`：失败记录 YAML（默认 `scripts/rehydrate_skipped.yml`）

---

## 第三步：`migrate_jekyll_to_hugo_book.py`（→ `content/docs`）

从 `_rehydrated_posts`（若为空则回退读 `_posts`）生成 Hugo 内容。

```bash
python3 scripts/migrate_jekyll_to_hugo_book.py
```

**注意**：

- 运行时会**删除并重建**整个 `content/docs/`。执行前请确认已备份或已提交 Git。
- 同一篇 `source_url` 在源数据里若对应多个文件，脚本会**合并为一篇**再输出。
- 若要从别的目录读入文章，可设置环境变量：  
  `WYGMJDD_POSTS_DIR=/绝对路径/到/markdown目录 python3 scripts/migrate_jekyll_to_hugo_book.py`

---

## 可选：`normalize_article_footer.py`（整理文末）

处理已生成在 Hugo 下的 Markdown（默认根目录为 `content/docs`）：

```bash
python3 scripts/normalize_article_footer.py
python3 scripts/normalize_article_footer.py --dry-run
python3 scripts/normalize_article_footer.py --root content/docs/2026/01
```

---

## 可选：`postprocess_image_captions.py`（图片说明样式）

把整行形式的 `![说明](url)图片下方说明文字` 转为 `<figure class="figure-with-caption">`，样式在 `assets/_custom.scss`。依赖 Hugo 配置里 Goldmark `unsafe = true`（本仓库已开）。

```bash
python3 scripts/postprocess_image_captions.py
python3 scripts/postprocess_image_captions.py --dry-run
python3 scripts/postprocess_image_captions.py content/docs/2026/01/summary__post-xxx.md
```

建议在第三步（及如有 `normalize_article_footer.py`）之后再跑，避免后续生成覆盖手工格式。

---

## 一键顺序参考（完整重来时）

```bash
cd /path/to/wygmjdd.github.io
source .venv/bin/activate

# 1. 占位
python3 scripts/migrate.py --stubs-only

# 2. 拉全文 + 图片
python3 scripts/rehydrate_posts.py

# 3. 生成 Hugo 正文（会清空 content/docs）
python3 scripts/migrate_jekyll_to_hugo_book.py

# 可选
python3 scripts/normalize_article_footer.py
python3 scripts/postprocess_image_captions.py

# 本地预览
hugo server -D
```

## 常见问题

1. **第二步很慢**：属正常；日志里会有进度与休眠提示。不要随意把 `--between-*`、`--sleep-*` 调得过小，以免触发风控。  
2. **图片仍打不开**：确认已 `playwright install`，且抓取逻辑会优先 `requests`、失败再用 Playwright。  
3. **侧栏分类不对**：改仓库根目录 `data/categories.yml`，与 `migrate_jekyll_to_hugo_book.py` 使用的数据源一致。  
4. **`content/docs` 被第三步清空**：迁移前务必备份或提交；仅改几篇时考虑只改单文件，而不是重跑第三步。

文档随仓库脚本变更可能过时；若参数与 `python3 scripts/某脚本.py --help` 不一致，以脚本为准。
