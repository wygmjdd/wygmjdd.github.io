# Content Transfer Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Migrate ~500 WeChat articles from album URLs into Jekyll `_posts`, then add time-based (year/month) and category-based (album as type) index pages; category URLs use English or pinyin slugs, display names stay Chinese.

**Architecture:** Python migration script reads `_data/wechat_albums.yml`, optionally parses album pages (Playwright) for article URLs or uses manual URL lists, fetches each article and converts to Markdown, writes `_posts` with slug-based categories; script or manual step builds `_data/categories.yml` (slug → Chinese name). Jekyll uses Liquid to build `/archive/` (year → month) and `/category/<slug>/` pages.

**Tech Stack:** Python 3 (requests, BeautifulSoup, html2text, Playwright, pypinyin), Jekyll, Liquid.

## Task 5: Migration pipeline (config → dedupe → fetch → write _posts)

**Files:**
- Create: `scripts/migrate.py` (main: read wechat_albums.yml, resolve slugs, get article URLs per album, dedupe by URL, for each URL fetch and write `_posts/YYYY-MM-DD-<slug>.md` with categories = list of slugs)
- Modify: `_config.yml` if needed so `_posts` and `_data` are used as standard Jekyll source

**Step 1:** Implement `migrate.py`: (1) load `_data/wechat_albums.yml`; (2) for each album compute slug (Task 2); (3) for each album get article URLs via `parse_album` or from manual URL list if present; (4) build url → list[slug] (one article can have multiple slugs); (5) for each URL (with rate limit and resume set): call `fetch_article`, then write `_posts/<date>-<filename_slug>.md` with front matter `title`, `date`, `categories: [slug1, ...]`, `source_url`; (6) ensure post filename slug is URL-safe and unique (e.g. title-based slug + optional index). Add a `--dry-run` and `--resume` (e.g. skip URLs already present in existing _posts or a done-list file).

**Step 2:** Generate or update `_data/categories.yml` from albums (slug → name) when writing posts, or in a separate step after migration.

**Step 3:** Manually run against one album (or a small manual URL list) and verify one or two generated posts in `_posts/`. Commit.

```bash

git commit -m "feat: migration pipeline from albums to _posts with categories"
```
